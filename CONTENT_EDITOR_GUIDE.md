# Руководство по редактору контента

## Описание

Редактор контента в админ-панели позволяет редактировать текстовое содержимое страниц сайта без необходимости изменения кода.

## Как использовать

### 1. Переход в редактор контента

1. Откройте админ-панель по адресу `/admin`
2. Нажмите на вкладку **"Редактор контента"**

### 2. Выбор страницы для редактирования

На главном экране редактора контента вы увидите список доступных страниц:
- **Главная** (home)
- **Доставка и оплата** (delivery)
- **Контакты** (contacts)

Нажмите на нужную страницу, чтобы начать редактирование.

### 3. Редактирование контента

После выбора страницы вы увидите все редактируемые секции:

#### Для главной страницы (Home):

**Hero секция:**
- Заголовок (поддерживается HTML, например: `<br />` для переноса строки)
- Подзаголовок

**Секция каталога:**
- Заголовок секции

**Секция услуг:**
- Заголовок секции
- 3 карточки услуг (название и описание для каждой)

#### Для страницы "Доставка и оплата":

**Hero секция:**
- Заголовок страницы

**Секции контента:**
- Каждая секция имеет:
  - Заголовок
  - Контент (поддерживается HTML)

#### Для страницы "Контакты":

**Hero секция:**
- Заголовок страницы

**Секции контента:**
- Адрес (контактная информация)
- Реквизиты (юридическая информация)

### 4. HTML в контенте

В полях контента вы можете использовать следующие HTML-теги:

- `<p>` - параграф
- `<br />` - перенос строки
- `<strong>` - жирный текст
- `<ol>` - нумерованный список
- `<ul>` - маркированный список
- `<li>` - элемент списка
- `<a href="...">` - ссылка

**Пример:**
```html
<p>Это параграф текста</p>
<ol>
  <li>Первый пункт</li>
  <li>Второй пункт</li>
</ol>
<p><a href="mailto:email@example.com">email@example.com</a></p>
```

### 5. Предпросмотр

Для секций с HTML-контентом доступен блок **"Предпросмотр"**, который показывает, как будет выглядеть отформатированный текст на сайте.

### 6. Сохранение изменений

1. После внесения изменений нажмите кнопку **"Сохранить изменения"**
2. Изменения сохраняются в localStorage браузера
3. Обновите нужную страницу сайта, чтобы увидеть изменения

### 7. Экспорт и сброс данных

**Экспорт контента:**
- Нажмите кнопку **"Экспорт контента (JSON)"** на главном экране редактора
- Будет скачан файл `content-YYYY-MM-DD.json` с текущим состоянием контента

**Сброс к начальным данным:**
- Нажмите кнопку **"Сбросить к начальным данным"**
- Все изменения будут отменены, и контент вернётся к исходному состоянию из `content.json`

## Техническая информация

### Структура данных

Контент хранится в файле `src/data/content.json` и в localStorage браузера.

Структура:
```json
{
  "pages": {
    "home": { ... },
    "delivery": { ... },
    "contacts": { ... }
  }
}
```

### Используемые файлы

- `src/data/content.json` - исходные данные контента
- `src/utils/contentManager.js` - утилиты для управления контентом
- `src/pages/Admin/AdminPanel.jsx` - компонент админ-панели с редактором
- Страницы, использующие контент:
  - `src/pages/Home/Home.jsx`
  - `src/pages/Delivery/Delivery.jsx`
  - `src/pages/Contacts/Contacts.jsx`

### Синхронизация изменений

Все изменения автоматически сохраняются в localStorage браузера. При обновлении страниц они автоматически подгружают актуальные данные из localStorage, если они есть, иначе используют данные из `content.json`.

## Добавление новых страниц

Чтобы добавить новую страницу в редактор контента:

1. Добавьте данные страницы в `src/data/content.json`:
```json
{
  "pages": {
    "newpage": {
      "id": "newpage",
      "title": "Новая страница",
      "hero": {
        "title": "ЗАГОЛОВОК"
      },
      "sections": [...]
    }
  }
}
```

2. Обновите компонент страницы, чтобы использовать `contentManager`:
```javascript
import { loadContentData, getPageById } from '../../utils/contentManager'

// В компоненте:
const contentData = loadContentData()
const pageData = getPageById(contentData, 'newpage')
```

3. Редактор автоматически подхватит новую страницу!
